1. CONFIGURAÇÃO DO application.properties
-------------------------------------------
ALTERAÇÃO: Configuração da conexão com MySQL
ARQUIVO: src/main/resources/application.properties

ALTERAÇÕES REALIZADAS:
- Configuração da URL de conexão com MySQL (jdbc:mysql://localhost:3306/eventos_academicos)
- Definição do usuário (root) e senha (vazia para XAMPP)
- Configuração do driver MySQL (com.mysql.cj.jdbc.Driver)
- Configuração do ddl-auto inicialmente como "update", depois alterado para "validate"

JUSTIFICATIVA:

  para "validate" para evitar alterações não intencionais no esquema existente
- show-sql ajuda a debugar queries SQL durante desenvolvimento

IMPACTO:
- Permite persistência de dados através de JPA/Hibernate
- Validação automática do esquema da base de dados
- Visibilidade das queries SQL geradas pelo Hibernate

5. CRIAÇÃO DE EXCEÇÕES CUSTOMIZADAS
------------------------------------
ALTERAÇÃO: Criação de classes de exceção customizadas
ARQUIVOS CRIADOS:
- src/main/java/com/eventos/apoio/exception/ResourceNotFoundException.java
- src/main/java/com/eventos/apoio/exception/BusinessException.java

ResourceNotFoundException:
- Exceção para quando um recurso não é encontrado pelo ID
- Construtor que aceita nome do recurso e ID
- Mensagem formatada: "{recurso} não encontrado com id: {id}"

BusinessException:
- Exceção para erros de regras de negócio
- Usada para violações de regras específicas do domínio
- Mensagem customizável

JUSTIFICATIVA:
- Tratamento diferenciado de diferentes tipos de erros
- Mensagens de erro mais claras e específicas
- Permite tratamento centralizado no GlobalExceptionHandler

IMPACTO:
- Separação clara entre erros de recurso e erros de negócio
- Melhor experiência do usuário com mensagens específicas
- Facilita o debugging e logging

IMPACTO:
- Todas as exceções são tratadas de forma consistente
- Respostas JSON padronizadas com mensagens claras

16. CRIAÇÃO DE DTOs (DATA TRANSFER OBJECTS)
---------------------------------------------
ALTERAÇÃO: Criação de DTOs para separar entidades da API
ARQUIVOS CRIADOS:
- src/main/java/com/eventos/apoio/dto/EventoDTO.java
- src/main/java/com/eventos/apoio/dto/ParticipanteDTO.java
- src/main/java/com/eventos/apoio/dto/InscricaoDTO.java
- src/main/java/com/eventos/apoio/dto/SessaoDTO.java
- src/main/java/com/eventos/apoio/dto/CertificadoDTO.java

EventoDTO:
- Campos: id (Integer), nome, descricao, dataInicio, dataFim, local, 
  capacidadeMaxima, estaAtivo
- Validações: @NotBlank, @Size, @NotNull, @Future, @Positive

ParticipanteDTO:
- Campos: id (Integer), nome, email, telefone, numeroEstudante, curso
- Validações: @NotBlank, @Size, @Email, @Pattern

InscricaoDTO:
- Campos: id (Integer), dataInscricao, estadoInscricao, participanteId, eventoId
- Validações: @NotNull
- IDs para relações em vez de objetos

SessaoDTO:
- Campos: id (Integer), titulo, nomeOrador, dataHoraInicio, dataHoraFim, eventoId
- Validações: @NotBlank, @Size, @NotNull

CertificadoDTO:
- Campos: id (Integer), codigoCertificado, dataEmissao, inscricaoId
- Validações: @NotBlank, @Size, @NotNull
- ID para relação em vez de objeto

JUSTIFICATIVA:
- Separar camada de API da camada de persistência
- Expor apenas dados necessários para o cliente
- Validação específica para dados de entrada
- Evitar exposição de informações sensíveis ou internas
- IDs para relações em vez de objetos para simplificar JSON

IMPACTO:
- Desacoplamento entre API e entidades
- Validação específica para dados de entrada
- JSON mais limpo e controlado
- Proteção contra modificação de campos internos
- Melhor manutenibilidade e testabilidade


17. CRIAÇÃO DE MAPPER CLASSES
------------------------------
ALTERAÇÃO: Criação de classes Mapper para conversão entre Entidades e DTOs
ARQUIVOS CRIADOS:

- src/main/java/com/eventos/apoio/mapper/ParticipanteMapper.java

ParticipanteMapper:
- toDTO(Participante): Converte entidade para DTO

SessaoMapper:
- toDTO(Sessao): Converte entidade para DTO
- toEntity(SessaoDTO): Converte DTO para entidade
- Injeção de EventoRepository
- Resolução de relação por ID (eventoId)
- @Component para injeção de dependência

CertificadoMapper:

- Injeção de InscricaoRepository
- Resolução de relação por ID (inscricaoId)
- @Component para injeção de dependência

JUSTIFICATIVA:
- Centralizar lógica de conversão entre entidades e DTOs
- Reutilização de código de conversão
- Separação de responsabilidades
- Resolução de relações por ID para evitar ciclos de serialização

23. ALTERAÇÃO DO DDL-AUTO PARA VALIDATE
-----------------------------------------
ALTERAÇÃO: Mudança de ddl-auto de "update" para "validate"
ARQUIVO: src/main/resources/application.properties

ALTERAÇÕES REALIZADAS:
- spring.jpa.hibernate.ddl-auto=update -> spring.jpa.hibernate.ddl-auto=validate

JUSTIFICATIVA:
- "update" tentava alterar o esquema automaticamente, causando erros
- "validate" apenas valida o esquema sem alterações
- Proteção contra alterações não intencionais na base de dados

IMPACTO:
- Validação do esquema sem alterações
- Proteção da estrutura da base de dados
- Evita erros de constraint durante startup
